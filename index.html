My Smart Home Project with Home Assistant

I am transforming my home into a smart living space using Home Assistant, an open-source automation platform. I run Home Assistant on a Raspberry Pi 3, which serves as the central hub for managing and automating my smart devices. My goal is to seamlessly integrate and control various devices in my two-story house, with a strong focus on automating my blinds and optimizing energy efficiency.

Technologies I Use

Raspberry Pi 3

The core of my smart home setup is a Raspberry Pi 3. It provides a lightweight yet powerful platform for running Home Assistant while ensuring flexibility for future upgrades. Despite its limited hardware resources, it handles automation tasks efficiently.

Zigbee Dongle

To create a reliable and energy-efficient smart home network, I use a Zigbee dongle. Zigbee's mesh capabilities allow me to connect and control devices such as sensors, lights, and other automation components without relying on Wi-Fi, making my setup more stable and responsive.

ACEMAGIC S1 Intel 12th Alder Lake N95/N97 Mini PC

I utilize a ACEMAGIC Mini PC for multiple things , since its connected trough etherenet, i can download torrents with a bigger speed , eliminating the time for downloading them trough wifi.

Challenges I Face

One of the biggest challenges I am facing is achieving precise control over my blinds. They use the UP-Jalousieaktor 1-Kanal 6A UPJ230/01 actuator, which requires careful calibration and compatibility adjustments within Home Assistant. I am actively testing different configurations to ensure smooth operation and responsiveness.

Future Plans

  • Enhance the reliability of the Acemagic Mini PC integration by refining automation rules and improving device response times.
  • Expand my smart home ecosystem by incorporating additional Zigbee devices, such as motion sensors, smart plugs, and security cameras.
  • Implement advanced automation scenarios, including voice control, adaptive shading based on indoor temperature, and geofencing for energy optimization.

This project is an exciting opportunity for me to deepen my understanding of IoT technologies and bridge different communication protocols into a unified home automation system.

My Local AI Playground with Ollama

I’ve set up a personal AI environment using Ollama, an open-source platform that allows you to run large language models locally on your machine without relying on the cloud. This setup gives me the freedom to experiment with powerful AI models in a private, offline, and customizable environment.

Technologies I Use

Ollama

Ollama is the core of my AI playground. It provides an easy way to download and run open-source AI models directly on my computer via a simple command-line interface and an accessible API. This lets me test different models and integrate them into custom applications or web interfaces securely and efficiently.

Local Web Interface

To make interacting with the models more user-friendly, I built a simple HTML-based web interface. It communicates with the Ollama API, sending prompts and displaying model responses in real-time, all while keeping data processing local on my machine. This ensures privacy and instant feedback without internet dependency.

Python Backend

Behind the scenes, a lightweight Python Flask server acts as a bridge between the web interface and the Ollama models. It handles API calls, manages model sessions, and routes responses back to the front end seamlessly.

Challenges I Face

One of the biggest challenges in this setup is managing hardware resources efficiently, especially when running larger models on devices with limited RAM and processing power. I’m currently optimizing my setup by testing different models and adjusting system configurations for better performance.

Future Plans

  • Add support for running multiple AI models in parallel for multitasking applications.
  • Enhance the web interface with features like prompt history, adjustable parameters (temperature, max tokens), and conversation memory.
  • Explore integrating voice input and text-to-speech output for a complete AI assistant experience.

This project is an exciting opportunity for me to explore the world of open-source AI tools, experiment with model fine-tuning, and build practical, private AI-powered applications tailored to my needs.

My Personal Password Manager in Python

I’m developing a secure and lightweight password manager using Python. The goal is to create a simple yet reliable application for storing and managing my passwords locally, without relying on third-party cloud services. Privacy, encryption, and ease of use are at the core of this project.

Technologies I Use

Python

The entire application is built with Python, taking advantage of its versatility and simplicity. Python makes it easy to implement file handling, encryption, and user interface components in a clean and maintainable way.

Cryptography Library

To keep my passwords secure, I use the cryptography library for strong encryption and decryption of sensitive data. It ensures that all stored credentials are encrypted with a unique key, making the password vault safe from unauthorized access.

Command Line Interface

For the user interface, I chose the Command Line, it's simple and easy for users to read and understand.

Challenges I Face

One of the biggest challenges in this project is securely managing encryption keys. The key needs to be both accessible for decrypting the vault and protected from unauthorized access. I’m currently working on a solution that involves deriving the key from a master password using a strong key derivation function (KDF) like PBKDF2 or scrypt.

Future Plans

  • Add a password generator tool with customizable rules for creating strong, unique passwords.
  • Implement a feature to export and import encrypted vaults for backup and recovery purposes.
  • Enhance the user interface with better styling and a search function for quickly finding stored credentials.
  • Consider integrating two-factor authentication (2FA) token storage and management in the future.

This project is a great hands-on opportunity to deepen my knowledge of Python security practices, encryption techniques, and GUI application development, while building a practical tool I can use in my daily life.

My Wi-Fi Password Recovery Tool in Python

I’m building a practical Python-based tool designed to retrieve saved Wi-Fi credentials from my own computer. The goal is to provide a simple, local utility to view and back up Wi-Fi profiles and passwords stored on my machine, useful for system recovery, network audits, or migrating to a new device.

Technologies I Use

Python

The core of the application is written in Python, making it lightweight, cross-platform (where possible), and easy to expand. It interacts with system-level commands to list Wi-Fi profiles and extract their saved credentials on authorized devices.

Subprocess and OS Modules

The tool uses Python’s subprocess and os modules to execute native system commands such as netsh wlan show profiles and netsh wlan show profile [name] key=clear on Windows systems, retrieving stored Wi-Fi passwords where permitted.

Features

  • List all saved Wi-Fi profiles on the system.
  • Display the stored password for each profile (if accessible).
  • Save retrieved profiles and passwords to a local encrypted file for backup.
  • Clear and simple command-line interface for ease of use.

Challenges I Face

The main challenge is ensuring cross-platform compatibility, since each operating system handles Wi-Fi credentials differently. I’m currently focusing on refining the Windows version and exploring solutions for MacOS and Linux support in future updates.

Future Plans

  • Add optional password export with AES encryption for secure backups.
  • Implement a Tkinter-based GUI for a more user-friendly interface.
  • Create logs for auditing retrieved data securely.

This project enhances my understanding of system-level scripting, process management, and local data security while providing a useful utility for personal IT management and system audits.
